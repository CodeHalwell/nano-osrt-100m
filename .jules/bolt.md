## 2024-05-19 - Vectorizing PyTorch Generation Loops for Speed and Batch Compatibility
**Learning:** In PyTorch, using python constructs such as `for token_id in set(generated[0].tolist()):` inside the autoregressive `generate` loop creates significant CPU overhead and fails when processing batches > 1 (hardcoded index 0). Also, using `.item()` to evaluate stopping conditions (e.g. `next_token.item() == eos_token_id`) raises a RuntimeError for batches.
**Action:** Always vectorize element-wise tensor logic in generation loops. I used boolean masking with `torch.where()` and `.scatter_()` to apply the repetition penalty, expanding the mask up to `max(vocab_size, generated.max().item() + 1)` and slicing back to ensure valid index bounds. For batch stopping conditions, use None-safe tensor equality: `(next_token == eos_token_id).all()`.
