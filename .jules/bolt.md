## 2025-05-02 - Vectorized Repetition Penalty in PyTorch Autoregressive Generation
**Learning:** PyTorch generation loops in HuggingFace-like models often apply `repetition_penalty` by iterating over `set(generated[0].tolist())` and manually scaling logits. For batch sizes > 1 or very long contexts, this Python loop and list conversion (`set(generated[b].tolist())`) becomes a massive performance bottleneck. In my profiling, a vectorized approach using `torch.gather()` and `torch.scatter_()` was ~460x faster than the nested Python loop approach for B=16, V=50277.
**Action:** When working on autoregressive generation code, replace Python-level iteration over tokens for applying penalties with vectorized combinations of `gather()`, `scatter_()`, and `torch.where()`. Crucially, out-of-bounds indices must be clamped or masked using `.masked_fill()` before gathering/scattering to prevent errors while applying correct token logic.
