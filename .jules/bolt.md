## 2024-05-24 - PyTorch Loop Performance in Autoregressive Generation
**Learning:** Found a Python list conversion `set(generated[0].tolist())` and Python loop within the autoregressive generation loop in `hf_model.py`. This is an extremely common PyTorch anti-pattern because it forces CUDA/CPU synchronization, prevents batching across the batch dimension (`0` was hardcoded!), and is O(N) in Python space. Using fully vectorized operations via boolean masking (`torch.where` + `scatter_`) prevents syncs and scales to N>1 batch size properly.
**Action:** Replace `tolist()` list manipulation with boolean mask-based tensor ops (`masked_fill`, `scatter_`, `where`) for element-wise modifications like repetition penalty in model.py or hf_model.py. Ensure batch dimension is fully utilized.
