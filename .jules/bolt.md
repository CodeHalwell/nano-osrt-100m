## 2024-05-24 - Optimizing Rotary Position Embedding (RoPE) application
**Learning:** In PyTorch, constructing intermediate full-size tensors before an element-wise operation uses significantly more memory bandwidth. For the RoPE function, `x_rot = torch.cat([-x2, x1], dim=-1)` followed by `x * cos + x_rot * sin` constructs an intermediate tensor `x_rot` of the same size as `x`, and then does two element-wise multiplications and an addition.
**Action:** By manually expanding the operation to `torch.cat([x1 * cos[..., :d] - x2 * sin[..., :d], x2 * cos[..., d:] + x1 * sin[..., d:]], dim=-1)`, we avoid creating `x_rot`, saving memory bandwidth and resulting in a ~1.4x speedup for the RoPE application without sacrificing correctness. This directly impacts both training and inference speeds.
