## 2025-02-14 - Python loop over tensor items causes generation crash
**Vulnerability:** DoS risk due to crash in generation loop with batch size > 1
**Learning:** Calling `.item()` on tensors or using Python loops and list conversions (like `set(generated[0].tolist())`) in autoregressive loops causes crashes or extreme bottlenecks when handling batches or when the code expects vectorized operations. Memory notes explicitly state: "In PyTorch generation loops, avoid using `.item()` on tensors to check stopping conditions, as it causes a RuntimeError for batch sizes > 1. Use vectorized checks with None-safety, such as `eos_token_id is not None and (next_token == eos_token_id).all()`." and "In PyTorch, avoid using Python loops and list conversions (like `set(tensor.tolist())`) for element-wise tensor modifications, especially in autoregressive generation loops. Always use vectorized operations like boolean masking, `.unique()`, and `torch.where()` to prevent significant performance bottlenecks."
**Prevention:** Avoid Python loops and `.item()` calls. Always vectorize operations like repetition penalty using `torch.where()` and `.unique()`.
